# Задание 1

##  Уровень 1
    _Mesto_ - проект фотобанка или специализированной соц сети, который имеет фунциональность загрузки фото, учета лайков и кастомизации профиля пользователя

### Общее по фреймфвормам

#### Single SPA
- Поддержка различных фреймворков: позволяет использовать несколько разных фреймворков одновременно (например, Angular, Vue.js, React).
Динамическая загрузка модулей: модули загружаются по мере необходимости, что улучшает производительность.
- Легкость интеграции: легко интегрировать существующие приложения в Single SPA.
- Гибкая архитектура: можно добавлять новые функции без изменения основной архитектуры.
- Сложность настройки: требует сложной конфигурации для правильной работы.
- Зависимость от роутинга: сильная зависимость от маршрутизации может усложнить управление приложением.

#### Webpack Module Federation

- Простота использования: относительно простая настройка по сравнению с другими решениями.
- Совместная работа компонентов: позволяет совместно использовать компоненты между различными приложениями.
- Высокая гибкость: поддерживает различные сценарии загрузки и совместного использования кода.
- Интеграция с существующими проектами: легко интегрируется с уже существующими проектами.
- Ограниченная поддержка фреймворков: изначально ориентирован на работу с Webpack, что ограничивает использование других инструментов сборки.
- Необходимость глубокой интеграции: требует глубокого понимания webpack-конфигурации.
- Проблемы с производительностью: при неправильной настройке могут возникнуть проблемы с производительностью.

В целом, оба фреймворка имеют значиимые плюсы и минусы. В данном  случае  использовать будем Webpack Federation Plugin, потому что всё приложение написано на React и его проще разделить используя этот подход.

## Уровень 2

Декомпозируем приложение по DDD:
- auth/  - Авторизация, аутентифиикация. Управление входом, регистрацией и сессиями	
- profile/ - 	Создание и редактирование профиля	
- photos/ - 	Загрузка и удаление фотографий	
- likes/ -	Учёт и отображение лайков	
- host/	Объединение микрофронтендов в единое приложение
